1. Introduction

1.1 Purpose

This software design specification is made with the purpose of outlining the software design and architecture of our project “Fitness365” in detail. This document will provide developers an insight in meeting client’s needs efficiently and effectively. Moreover, this SDS document facilitates communication and understanding of the system by providing several views of the system design.

2. Document Conventions

5. Class Diagram

5.1 Detailed Class Diagrams


This section focuses on the high level class diagram which demonstrates the relation between the classes. A detailed description of each class is given for a clear grasp of their function in the product.

5.2 Detailed Class Description


  • User:
    • This is an abstract class and it stores all the information of the users (both admin and member). It is then extended by admin and member to add their exclusive property and funcions.
  • Admin:
    • This class extends the user class. It can register user, set workout routine, diet chart, view transactions and many more.
  • Member
    • This class extends the user class. It can perform functions like pay monthly fee, submit attendance etc.
  • Attendance
    • It stores user attendance. It can generate attendance pattern checker.
  • Diet Chart
    • It stores all the information of the member's dietchart.
  • Work Out Routine
    • It stores all the information of the member workout routine.
  • Payment Record
    • It stores the user payment record and updates that in the database.
  • Transaction
    • It calculates the gym's net profit every month and reports to the admin.
